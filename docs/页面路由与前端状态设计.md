# 页面路由与前端状态设计

## 1. 文档目标

本文档定义当前 Demo 的页面结构、导航边界与前端状态真值。

---

## 2. 路由结构

当前固定路由：

```text
/jobs
/jobs/new
/jobs/new/from-description
/jobs/new/from-form
/jobs/[jobId]
/jobs/[jobId]/edit
/jobs/[jobId]/candidates/new
/candidates/[candidateId]
```

这些路由对应当前 Demo 的完整业务闭环，不再保留旧页面分支。

---

## 3. 状态设计原则

### 3.1 服务端状态

以下状态属于服务端状态，由 React Query 管理：

- Job 列表
- Job 详情
- Job 编辑初始化数据
- Candidate 列表
- Candidate 详情
- Candidate 导入上下文
- AnalysisRun 状态

### 3.2 页面本地状态

以下状态只存在于页面本地：

- Job 编辑区当前值
- AI 对话输入
- Candidate 导入页原始文本
- 已选择文件
- Candidate 列表筛选与搜索
- 处理状态区临时输入

### 3.3 URL 规则

当前 Demo 中，Job 详情页 Candidate 列表的筛选、排序、标签和搜索都不进入 URL。  
它们属于页面本地状态，刷新页面后恢复默认值。

---

## 4. 页面真值

### 4.1 `/jobs`

- 展示全部已生效 Job
- 不展示 `draft`
- 右上角入口固定为“添加岗位”

### 4.2 `/jobs/new`

- 只作为创建入口页
- 跳转到：
  - 已有描述导入
  - 基础信息生成

### 4.3 `/jobs/new/from-description`

- 输入已有岗位描述
- 点击“生成”创建 Job draft
- 成功后跳转 `/jobs/[jobId]/edit`

### 4.4 `/jobs/new/from-form`

- 输入结构化基础信息
- 点击“生成”创建 Job draft
- 成功后跳转 `/jobs/[jobId]/edit`

### 4.5 `/jobs/[jobId]/edit`

这是 Job 编辑核心页。

页面固定分为三块：

- 岗位定义编辑区
- rubric 编辑区
- AI 交互区

底部操作固定为：

- 保存
- 取消

#### 保存语义

- 不直接入库
- 先创建 `job_finalize` analysis run
- 显示统一进度卡
- run 成功后跳转岗位详情

#### AI 区语义

- 获取建议：只追加对话
- 生成新版：覆盖当前编辑态

### 4.6 `/jobs/[jobId]`

岗位详情页展示：

- 岗位基础信息
- 当前 rubric
- Candidate 列表

Candidate 列表筛选为页面本地状态：

- 搜索
- 排序
- 状态筛选
- 标签筛选

这些交互不能触发 URL 变化，也不能打断输入焦点。

### 4.7 `/jobs/[jobId]/candidates/new`

Candidate 分析入口页。

页面固定包括：

- 岗位上下文
- 原始文本输入
- PDF 上传区
- Candidate 分析进度卡

点击“分析”后：

- 创建 `candidate_import` analysis run
- 通过 `SSE` 更新进度
- run 成功后跳转 Candidate 详情页

### 4.8 `/candidates/[candidateId]`

Candidate 详情页固定顺序：

1. 标准化信息
2. 原始输入
3. 汇总结论
4. 逐项分析
5. 处理状态

原始文件区只展示：

- 文件名
- 类型
- 页数
- 查看原文件链接

不展示：

- 提取文本
- 服务器路径

---

## 5. AnalysisRun 前端语义

前端统一使用：

- `startJobFinalizeRun`
- `startCandidateImportRun`
- `GET /api/analysis-runs/{run_id}`
- `GET /api/analysis-runs/{run_id}/events`

前端统一消费四类 SSE 事件：

- `connected`
- `progress`
- `completed`
- `failed`

进度卡统一展示：

- 当前阶段
- `currentAiStep / totalAiSteps`
- 错误信息

---

## 6. 当前固定结论

- Job 保存与 Candidate 分析都不是同步长请求
- Candidate 列表筛选不再进入 URL
- Candidate 详情页原始文件只看原 PDF
- Candidate 详情页布局顺序已经固定
