# Demo 设计文档

## 1. 产品定位

本 Demo 不是完整 ATS，也不覆盖职位发布和候选人抓取。  
它聚焦于招聘后处理链路：

**岗位准备 → 候选人资料分析 → AI 初筛展示 → 人工处理与后续沟通**

---

## 2. 核心页面

当前 Demo 固定包含以下页面：

- 岗位列表页
- 岗位新建入口页
- 岗位描述导入页
- 岗位基础信息生成页
- 岗位编辑页
- 岗位详情页
- Candidate 分析页
- Candidate 详情页

---

## 3. 岗位编辑页

岗位编辑页是岗位创建与修改的核心页面。

### 3.1 页面结构

- 岗位定义编辑区
- rubric 编辑区
- AI 对话区
- 底部操作区

### 3.2 AI 交互

当前固定为两种模式：

- 获取建议
  - 只返回建议文本
  - 不覆盖当前编辑区
- 生成新版
  - 直接覆盖当前编辑区
  - 更新标题、摘要、描述、responsibilities、skills 和 rubric

### 3.3 底部操作

底部固定只有两个按钮：

- 保存
- 取消

保存语义：

- 不直接同步入库
- 先启动 `job_finalize` analysis run
- 显示进度卡
- 成功后进入岗位详情页

取消语义：

- 新建流程取消时删除对应 `draft`
- 已生效岗位编辑取消时直接返回详情页

---

## 4. 岗位详情页

岗位详情页展示：

- 岗位标题与摘要
- 岗位正文与结构化信息
- 当前 rubric
- Candidate 列表

Candidate 列表支持：

- 搜索
- 排序
- 状态筛选
- 标签筛选

这些交互都是页面内响应式操作，不写入 URL。

---

## 5. Candidate 分析页

Candidate 分析页负责提交：

- 原始文本
- PDF 文件

按钮文案固定为“分析”。

点击后：

1. 创建 `candidate_import` analysis run
2. 页面显示进度卡
3. 通过 `SSE` 实时更新进度
4. 完成后跳转 Candidate 详情页

---

## 6. Candidate 详情页

Candidate 详情页固定顺序如下：

1. 标准化信息
2. 原始输入
3. 汇总结论
4. 逐项分析
5. 处理状态

### 6.1 原始输入

- 原始文本直接展示
- 原始文件只展示 PDF 文件信息和“查看原文件”链接
- 点击后在新标签页打开原 PDF

### 6.2 汇总结论

展示：

- AI summary
- 推荐结论
- 默认 AI 标签与人工标签
- 硬门槛总体结果

不再展示：

- 5 分总分

### 6.3 处理状态

当前支持：

- 修改状态
- 添加人工标签
- 添加备注
- 生成邮件草稿

---

## 7. Candidate 分析语义

Candidate 是否入库只取决于：

- 分析链路成功
- 资料被识别为有效候选人资料
- 存在有效姓名

硬门槛结果不阻断入库，只影响：

- 默认标签
- 列表筛选
- 详情页展示
