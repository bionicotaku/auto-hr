# 工程实现文档

## 1. 文档目标

本文档定义当前 Demo 的工程真值，用于统一前后端、数据库、AI 工作流和页面行为。

当前系统只服务于这条闭环：

**Job 创建与编辑 → Job 保存定稿 → Candidate 分析 → Candidate 展示 → 状态/标签/备注/邮件草稿**

本文档只描述当前实现，不保留旧方案。

---

## 2. 当前固定方案

### 2.1 长耗时链路

所有长耗时 AI 主链路统一走：

- `AnalysisRun` 持久化状态快照
- `AnalysisRunEvent` 持久化事件流
- `SSE` 向前端推送进度
- API 进程内后台执行

当前接入的 run 只有两类：

- `job_finalize`
- `candidate_import`

### 2.2 Job 真值

- `/jobs` 只展示 `active` Job，不展示 `draft`
- 新建岗位首次生成后先创建 `draft`
- 编辑页点击“保存”时，不直接同步入库，而是启动 `job_finalize` analysis run
- `job_finalize` 成功后，当前 Job 生效内容被更新
- 新建流程点击“取消”会删除对应 `draft`
- 已生效岗位进入编辑页后：
  - 无修改时点击“保存”等同返回详情页
  - 有修改时点击“保存”会启动 `job_finalize` analysis run

### 2.3 Candidate 真值

- Candidate 导入入口页按钮文案是“分析”
- Candidate 分析走 `candidate_import` analysis run
- 只有分析链路成功、标准化确认是有效候选人资料、且提取出有效姓名时才入库
- `hard_requirement_overall` 与 `recommendation` 不阻断入库
- `hard_requirement_overall` 只影响默认 AI 标签与展示语义
- 默认标签规则：
  - `has_fail` → `硬性要求未通过`
  - `has_borderline` → `需要复核`
- Candidate 不再保留旧的 5 分总分字段

### 2.4 Candidate 原始文件真值

- 系统保留原始 PDF 文件
- 不提取、不保存、不展示 PDF 文本内容
- Candidate 详情页“原始文件”区域只展示文件元数据和 `file_url`
- 文件通过后端接口以浏览器可直接打开的 PDF 形式访问
- 正式文件名格式固定为：`[候选人姓名]-[标号].pdf`

### 2.5 Candidate 列表交互真值

- Job 详情页中的 Candidate 列表筛选、排序、搜索都属于页面本地状态
- 这些条件不写入 URL
- 搜索输入本地即时响应，请求值可做轻量 debounce
- 切换排序、状态、标签、搜索时不触发导航

## 5. 页面到后端的主链路

### 5.1 岗位创建与编辑

1. 前端创建 Job draft
2. 跳转 `/jobs/[jobId]/edit`
3. 编辑页本地维护：
   - title
   - summary
   - description_text
   - responsibilities
   - skills
   - rubric_items
4. 获取建议只返回对话文本
5. 生成新版会覆盖当前编辑态
6. 点击“保存”时：
   - 创建 `job_finalize` run
   - 通过 SSE 展示进度
   - 完成后跳转 `/jobs/[jobId]`

### 5.2 Candidate 分析

1. 前端在 `/jobs/[jobId]/candidates/new` 提交原始文本和 PDF
2. 后端创建 `candidate_import` run
3. 前端通过 SSE 展示进度
4. 后端依次执行：
   - 输入准备
   - Candidate 标准化
   - 有效性判断
   - rubric 逐项分析
   - supervisor 汇总
   - 成功后入库
5. 完成后跳转 `/candidates/[candidateId]`

### 5.3 Candidate 详情处理

Candidate 详情页展示 5 块内容：

1. 标准化信息
2. 原始输入
3. 汇总结论
4. 逐项分析
5. 处理状态

处理状态区支持：

- 更新状态
- 添加人工标签
- 添加备注
- 生成邮件草稿

## 6. 数据与文件

### 6.1 数据库

迁移历史已经压平成单条初始链：

- `20260410_0007_initial_schema`

当前数据库只保留当前真值结构，不保留中间过渡 migration。

### 6.2 文件

- Candidate 上传文件先落临时目录
- 分析成功后移动到正式目录
- 正式文件名按候选人姓名重命名
- Candidate 详情页通过后端文件接口访问 PDF
- 前端不直接读取服务器路径

---

## 7. 验收基线

当前工程文档与实现必须同时满足：

- Job 保存与 Candidate 分析都通过 `AnalysisRun + SSE`
- Candidate 原始文件只展示 PDF 访问链接，不展示提取文本
- Candidate 入库不受硬门槛与推荐结论冲突影响
- Candidate 列表筛选不再由 URL 驱动
- 数据模型、API、页面行为、AI 输出语义在各文档中保持一致
