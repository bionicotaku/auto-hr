# 前端视觉重构实施方案

## 1. 背景与目标

本次任务是把 `apps/web` 的真实业务前端，全面重构为接近 `/ui-lab` 的视觉体系。

固定目标：

- 保持当前业务功能、路由、接口和交互语义不变
- 允许页面内部布局与模块编排彻底重构
- 不做视觉层面的向后兼容
- 真实业务页面以中文为主
- `/ui-lab` 保留，作为视觉参考页
- 真实业务页面支持 light / dark，两套主题都交付
- 真实业务页面每次跟随系统主题，不保存用户主题偏好

## 2. 当前真值约束

- 前端主链路保持为：岗位列表 -> 新建岗位 -> 岗位编辑 -> 岗位保存 -> Candidate 分析 -> Candidate 详情 -> 状态/标签/备注/邮件草稿
- Candidate 详情页固定顺序不能改：
  1. 标准化信息
  2. 原始输入
  3. 汇总结论
  4. 逐项分析
  5. 处理状态
- Job 详情页 Candidate 列表筛选、排序、搜索都是页面本地状态，不进入 URL
- Job 保存和 Candidate 分析仍然统一走 `AnalysisRun + SSE`
- 不展示 PDF 提取文本，不暴露服务器文件路径

## 3. 视觉目标与非目标

视觉目标：

- 明亮背景上的 glass surface
- 更统一的圆角、边框、阴影和层级
- 更稳定的 hover 与 focus 反馈
- 业务页整体更接近 `/ui-lab` 的 section / panel / review surface 节奏
- dark mode 不只是简单反色，而是完整主题 token

非目标：

- 不把 `/ui-lab` 的英文样张文案直接搬进业务页
- 不为了视觉重构而修改业务流程
- 不把 `/ui-lab` 强行变成真实页面的实现来源

## 4. 主题策略

- 真实业务页默认跟随系统主题
- 右下角统一悬浮 sidebar 提供手动浅色 / 深色切换
- 手动切换只在当前会话内生效，刷新后重新回到系统主题
- 不使用 localStorage 保存主题偏好
- `/ui-lab` 可继续保留自己的本地主题切换作为样张能力
- 所有真实业务页统一依赖全局 theme tokens，而不是局部硬编码亮暗样式

## 5. 共享组件与布局层设计

本次重构先从共享层入手，统一以下能力：

- `globals.css` 中的 light / dark token
- `AppShell` 的业务页外壳
- `components/ui/` 内的 Button / Card / Input / Select / Textarea / Badge / Spinner / ErrorStateCard / AnalysisProgressCard
- 必要时补充统一的 section header / toolbar / review row shell 这类展示模式

规则：

- 共享层不接业务 API
- 共享层只负责视觉一致性、交互反馈和布局模式
- 业务页围绕共享层组织，不再逐页散落写视觉规则

## 6. 各业务页面改造方案

### 6.1 Jobs 相关页面

- `/jobs`：重构为更强的 overview + review surfaces 结构
- `/jobs/new`：入口卡片统一成 glass surface 模式
- `/jobs/new/from-description`：重构为更强的单页 form workspace
- `/jobs/new/from-form`：重构为分组表单工作区
- `/jobs/[jobId]`：让 Candidate 列表成为主 review surface，吸收 `/ui-lab` 中 `Candidate signal table` 的视觉方向

### 6.2 Job 编辑页

- 重构为 definition / rubric / AI chat / action bar 的统一 workspace
- 保留当前本地编辑态、chat、agent-edit、finalize、cancel 行为

### 6.3 Candidate 页面

- `/jobs/[jobId]/candidates/new`：重构为双栏导入工作区
- `/candidates/[candidateId]`：维持既定顺序，统一到 review panels 体系
- 处理状态区重构为更清晰的 controls + history 组合

## 10. 风险与回滚边界

- 本次不做视觉向后兼容，因此页面外观会整体替换
- 若某页改造影响测试或语义，需要优先回到当前业务真值，而不是坚持样张外观
- dark mode 的实现以系统主题跟随为准，不引入新的用户偏好状态
